# Activity Ledger

Use the ledger for append-oriented historical memory.

Ledger entries should answer:

- what happened;
- when it happened;
- why it mattered;
- what was decided;
- what worked;
- what failed;
- where supporting artifacts live.

Prefer monthly files named `YYYY-MM.md` in the local durable-notes hierarchy.
Link to issues, PRs, commits, logs, and runbooks instead of duplicating long
content.

Do not use the shared repo as a live activity ledger. Each agent should maintain
its own local ledger after merging these notes.
